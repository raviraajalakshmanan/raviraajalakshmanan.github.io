### [Learn-Offline](https://github.com/raviraajalakshmanan/Learn-Offline)

**Python / CLI**

A responsible offline learning utility for people who have paid for their courses and would like to watch them somewhere their internet connection cannot emotionally disappoint them.

Learn-Offline wraps `yt-dlp` and the Udemy API to batch-download owned course content with subtitle extraction, quality selection, and chapter-organized folders. It supports macOS, Linux, and Windows, and can extract cookies from common browsers to make authentication less painful.

Built because “I’ll watch it later” should still work on a plane.

[GitHub](https://github.com/raviraajalakshmanan/Learn-Offline)

### [NiralCPP](https://github.com/raviraajalakshmanan/NiralCPP)

**C++ / Algorithms**

A design-driven C++ reference for algorithms and data structures, built for quick revision and clean implementation. Every file compiles standalone with a working driver, uses C++17, follows LLVM-style formatting, and avoids unnecessary project scaffolding. Just `g++`, the implementation, and the problem in front of you.

The collection includes graph algorithms such as BFS, DFS, Dijkstra, Bellman-Ford, Kruskal, Prim, and Union-Find, along with tree traversals and other commonly used data structures. It grows whenever I need to re-learn something properly instead of trusting a random competitive programming snippet with five macros and a dream.

[GitHub](https://github.com/raviraajalakshmanan/NiralCPP)

### [OCR Text Extractor](https://github.com/raviraajalakshmanan/OCR-Text-Extractor)

**Python / AWS / AI**

An OCR pipeline for converting owned reading material into clean, editable text. The tool automates screenshots from Kindle, Apple Books, and PDF viewers, handles double-page layouts, and processes the output through AWS Textract. It can also use Amazon Bedrock with Claude 3.5 Sonnet to clean up OCR output into structured Markdown.

The goal is simple: if you legally own the material, you should be able to study, search, and annotate it in the tools you actually use. Also, manually typing pages from a technical book feels like something we invented computers specifically to avoid.

[GitHub](https://github.com/raviraajalakshmanan/OCR-Text-Extractor)
